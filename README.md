# やまのぼり学習スケジュール 🏔️

1日のスケジュール管理と「実際に行動した実績」の記録をサポートする、個人専用のスマホ向けWebアプリ（Phase 1 MVP）。

`index.html` 1ファイルだけで動きます。**ビルド不要・サーバー不要**。データは端末のブラウザ内（localStorage）にのみ保存されます。

## 実装済み機能（Phase 1）

- **今日のスケジュール画面** … 完了タスクは即非表示。残りをコンパクトに一覧。
- **実績タイマー** … 各タスクを「開始／終了」で計測。押し忘れても開始・終了時刻を後から手動修正可。
- **自動スライド** … 実績に応じて未完了タスクの時刻を前後どちらにも詰め直し。
- **固定ブロック回避** … 「学校」などの固定予定にはタスクを割り込ませず、終了後に配置。
- **24時リミット** … 設定した時刻（既定24:00、26:00等に変更可）を超えるタスクは短縮／消滅。
- **セットメニュー** … 「平日通常／英検重視／英語重視／世界史重視」を一括展開。設定から編集可。
- **今日の自己評価** … ★入力＋ユーモアのある定型フィードバック。

## ローカルで開く

`index.html` をブラウザでそのまま開くだけです（ダブルクリックでOK）。

## GitHub Pages で公開してスマホで使う

1. GitHub で新しいリポジトリを作成。
2. このフォルダの中身（`index.html`, `README.md`）をアップロード（またはコミット＆プッシュ）。
   ```bash
   git init
   git add .
   git commit -m "Phase 1 MVP"
   git branch -M main
   git remote add origin https://github.com/<あなたのID>/<リポジトリ名>.git
   git push -u origin main
   ```
3. リポジトリの **Settings → Pages → Build and deployment** で、Source を `Deploy from a branch`、Branch を `main` / `/(root)` に設定して保存。
4. 数十秒後に `https://<あなたのID>.github.io/<リポジトリ名>/` でアクセス可能に。
5. スマホのブラウザでそのURLを開き、「ホーム画面に追加」すればアプリのように使えます。

> データは開いた端末ごとに保存されます（端末間の同期は Phase 2 以降でクラウド対応予定）。
